## Key Visual Elements

### Floor Selection Hint (NEW)
- **Background**: Light blue with transparency `rgba(33, 150, 243, 0.15)`
- **Border**: 4px solid blue on left side (accent color)
- **Text Color**: Blue (accent primary)
- **Font Size**: 0.9em
- **Padding**: Medium spacing
- **Border Radius**: Small rounded corners

### Behavior Differences

| Component Type | Hint Shown? | Warning (‚ö†Ô∏è)? | Checkboxes Auto-Selected | User Must Select |
|---------------|-------------|---------------|-------------------------|------------------|
| Multi-floor required | Yes (blue) | Yes (yellow) | Min floors needed | ‚â• Required floors |
| Large item | Yes (blue) | No | Only floor 1 | ‚â• 1 floor |
| Normal | No | No | N/A (dropdown) | 1 floor |

## Color Scheme

- **Hint Box Background**: `rgba(33, 150, 243, 0.15)` (semi-transparent blue)
- **Hint Border**: `var(--color-accent-primary)` (cyan/blue #00d4ff)
- **Hint Text**: `var(--color-accent-primary)` (cyan/blue #00d4ff)
- **Warning Text**: `var(--color-warning)` (yellow #ffc107)

## Interaction Flow

### For Large Items (25-100%)

1. User clicks component (e.g., Staterooms 560t)
2. Modal opens with:
   - Blue hint message suggesting multi-floor
   - Only Floor 1 checked
   - No warning notice
3. User can:
   - Keep 1 floor (will fit but be very large)
   - Check additional floors (will split component)
4. Click "Place Component"
5. Validation: At least 1 floor must be selected

### For Required Multi-Floor (>100%)

1. User clicks component (e.g., Fuel Tank 2000t)
2. Modal opens with:
   - Blue hint showing required floor count
   - Yellow warning notice
   - Required floors auto-checked
3. User can:
   - Keep minimum floors
   - Add more floors (will spread component more)
4. Click "Place Component"
5. Validation: At least required floors must be selected

## Accessibility Notes

- Color is not the only indicator (also uses icons: ‚ÑπÔ∏è, ‚ö†Ô∏è)
- Messages use clear, descriptive text
- Visual hierarchy: hint ‚Üí warning ‚Üí checkboxes
- Checkbox labels are clickable
- Clear validation messages if user tries to proceed incorrectly
